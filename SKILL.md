# 🔔 auto-skill-scout

당신의 지난 한 주 대화를 분석해서 **막혔던 문제 + 맞춤 스킬 추천**을 **이미지 UI 주간 보고서 한 장**으로 보내주는 스킬입니다.
마치 개인 보고서처럼 당신의 작업을 판단하고, 필요한 스킬을 자동으로 추천합니다.

## 🎯 핵심 기능

- 📅 **주 1회 자동 실행** — 매주 월요일 오전, 지난 한 주 분석 & 추천
- 📊 **주간 사용 브리프** — 반복해서 막혔던 문제 3가지 (당신이 한 말 그대로 인용)
- 🎯 **맞춤 스킬 추천** — 2~3개만 (Before/After + GitHub 링크)
- 🖼️ **이미지 UI 보고서** — 텍스트 나열이 아닌, 한눈에 보는 카드형 시각 보고서
- 💾 **자동 저장** — `_scout-recommendations.md`에 매주 기록
- 🔔 **Claude 알림** — 따로 설정 필요 없음

## 📖 사용 방법

### 설치 (터미널 필요 없음)
Claude Code를 열고 아래를 그대로 붙여넣으세요:
```
이 스킬 설치하고 매주 월요일 오전 9시에 실행되게 해줘:
https://github.com/Koo-bon/auto-skill-scout
```
Claude가 다운로드 → 설치 → 스케줄 설정까지 해줍니다. "허용"만 누르면 끝.
(첫 실행 때 도구를 허용하면, 다음 주부터는 안 물어봅니다.)

**주기·시간 변경** — 말로 하면 됩니다:
```
auto-skill-scout를 매주 금요일 오후 6시로 바꿔줘
```

### 완료! 🎉
- 매주 월요일 오전, Claude에서 이미지 UI 보고서 알림
- 막혔던 문제 3가지 + 추천 스킬 2~3개 (Before/After + GitHub 링크)
- `_scout-recommendations.md`에 자동 저장

> ⚠️ 스케줄은 **Claude 앱이 열려 있을 때** 실행됩니다. 예약 시간에 앱이 꺼져
> 있었다면 **다음에 앱을 켤 때** 실행됩니다. (알림을 놓치지 않습니다.)

## ⚡ 실행 규칙 — 속도 상한 (반드시 지킬 것)

이 스킬이 실행될 때는 아래 상한을 지켜 **2분 이내**로 끝낸다. 툴 호출을 최소화한다.

1. **대화 분석** — 세션 목록 조회 1회 + 트랜스크립트 검색 **최대 2회**. 그 이상 검색 금지.
2. **스킬 검색** — WebSearch **최대 2회**, 추천은 **2개만**. 서버 오류(529 등)가 나면
   **재시도하지 말고** "이번 주 검색 실패"로 표시하고 넘어간다. 이미 보유한 내장 스킬이면
   검색 없이 "설치 불필요"로 추천.
3. **이미지 보고서** — 고정 템플릿으로 **한 번에** HTML 작성 후 **Artifact 1회 게시**.
   재작성·재게시 금지.
4. **금지** — WebSearch 재시도, 3회 이상 검색, 추천 3개 이상, 코드 검증(verify)·추가 폴리싱.

> 💡 **왜?** 실행이 오래 걸리는 주범은 ① 검색 서버 일시 과부하로 인한 재시도 ②
> 과도한 세션 검색이다. 위 상한이 둘 다 차단한다. 또한 이 작업은 **백그라운드**에서
> 돌아가므로 사용자가 옆에서 기다릴 필요가 없다 — 끝나면 보고서+알림만 남는다.

> 🔐 **첫 실행 승인 안내** — 설치한 사람은 **첫 주 1회** 도구 사용(WebSearch·세션 분석·
> 보고서 발행)을 허용해야 한다. 이 승인은 각자 자기 기기에 저장되며, 한 번 허용하면
> 다음 주부터는 묻지 않는다. (Claude Code의 보안 기능이라 스킬이 끌 수 없음.)

> 🧩 **환경 호환** — 세션 검색 도구(`mcp__ccd_session_mgmt__*`)가 없는 환경이면,
> 최근 git log·파일 변경으로 대체 분석하고 그 사실을 보고서에 한 줄 명시한다.

## 🔍 "정말 내 패턴을 이해할까?" — 신뢰도 확보

### ✅ 구체적 사례로 확인하세요

**예: 월요일 09:00 추천이 들어옵니다**

```
당신이 지난주에 한 실제 요청들:
┌─ 월요일 09:15: "이미지 여백 자동으로 제거할 도구 없을까?"
│  시간: 오전 10:30 (예정)
│  빈도: 지난 3일간 5회
│
├─ 수요일 14:00: "팀이 5명이 되면서 협업 도구가 필요해"
│  빈도: 지난주 대비 3배 증가
│
└─ 금요일 16:30: "README 작성을 좀 더 깔끔하게 하고 싶은데..."
   빈도: 주 3회

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

오늘 추천:

1️⃣ jezweb image-processing
   → "월요일 9:15 당신의 요청" + "오전 10-11시 패턴" 감지 ✓

2️⃣ oh-my-claudecode
   → "팀 협업 3배 증가" 패턴 감지 ✓

3️⃣ oh-my-claudecode:writer
   → "README 작성" 요청 + 주 3회 패턴 감지 ✓
```

**이렇게 작동합니다:**
- 📍 **시간**: 지난 3일 내 당신의 활동 시간대
- 🗣️ **실제 요청**: Claude와 나눈 대화 키워드
- 💡 **추천 이유**: 그 요청을 푸는 검증된 스킬

→ **"아, 이건 정말 내 상황을 이해했구나"** 신뢰도 ⬆️

더 자세히: **[Before & After 사용 예시 보기](BEFORE_AFTER.md)** 
— 이 스킬을 사용했을 때 당신의 업무가 어떻게 달라지는지 확인하세요.

## 📁 자동 생성 파일

### `_scout-recommendations.md`
매일의 추천 기록이 자동으로 저장됩니다.
시간이 지나면서 당신의 추천 히스토리를 확인할 수 있습니다.

## 🔄 작동 흐름

```
설정된 시간 (예: 매일 09:00)
    ↓
당신의 대화 자동 분석 (지난 7일)
    ↓
작업 패턴 추출 + 브리프 작성
    ↓
Reddit/GitHub에서 맞춤 스킬 검색
    ↓
최고 검증된 2~3개 선별
    ↓
Claude 알림으로 전송
    ↓
파일에 자동 저장
```

## ✨ 특징

| 항목 | 설명 |
|------|------|
| **설정** | `/schedule` 한 줄로 완료 |
| **알림** | Claude Code 자체 알림 (추가 설정 없음) |
| **추천** | 2~3개만 (간결하고 선택 쉬움) |
| **검증** | Reddit/GitHub 인기도 기반 |
| **저장** | 자동으로 md 파일에 기록 |
| **관리자** | 필요 없음 (개인 사용) |

## ❓ FAQ

**Q: 알림이 안 와요**
A: `/schedule list` 로 스케줄 확인 후, 설정 시간이 지났는지 확인하세요.

**Q: 시간을 바꾸고 싶으면?**
A: `/schedule "auto-skill-scout" --daily 14:00` 등으로 재설정하세요.

**Q: 지난 추천들은?**
A: `_scout-recommendations.md` 파일에 모두 저장되어 있습니다.
